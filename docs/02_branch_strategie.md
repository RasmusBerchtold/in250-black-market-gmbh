### Git Strategie: Github Flow

## Main Branch
Der `main` Branch hat immer den Aktuellen code und wird nicht verwendet zum bearbeiten 

## Feature Branch

der `feat/Branch` wird verwendet um jede neue Funktion sowie Bugfix (mit `hotfix/...`) der von main abzweigt zu bearbeiten. 

## Pull Request
Hier werden Änderungen die in Branches sind wieder in den `main` geführt. Ein PR (Pull Request) muss mindestens von einem anderen Mitarbeiter (oder sich selbst)geprüft werden. 

## Realses 
Sobald der `main` einen stabilen Stand hat wird er anahnd eines Tags mit Versionen beschriftet.
